1. 用户访问前端应用 (React + TypeScript + Tailwind CSS)
   - 部署在 Vercel 上
   - 提供游戏大厅、游戏页面、汤底页面等界面

2. 前端通过 HTTP 请求与后端通信
   - RESTful API 或 GraphQL
   - 请求类型：开始游戏、提问、查看汤底、用户认证等

3. 后端处理业务逻辑 (Node.js + Express)
   - 接收前端请求
   - 管理游戏会话状态
   - 处理用户认证和授权
   - 与数据库交互（存储/检索数据）

4. 后端调用 AI 服务 API (DeepSeek)
   - 发送用户提问和游戏上下文
   - 接收 AI 的"是/否/无关"回答
   - 处理 AI 生成的故事内容

5. 数据持久化层
   - PostgreSQL 或 MongoDB 数据库
   - 存储用户信息、游戏记录、故事库等

6. 响应返回路径
   - AI 服务 → 后端 → 前端 → 用户界面

1. 游戏开始流程：
   前端 → 后端: POST /api/game/start {storyId}
   后端 → 数据库: 查询故事详情
   后端 → 前端: 返回游戏会话ID和汤面故事

2. 提问回答流程：
   前端 → 后端: POST /api/game/ask {sessionId, question}
   后端 → AI服务: 发送上下文和问题
   AI服务 → 后端: 返回"是/否/无关"回答
   后端 → 数据库: 保存对话记录
   后端 → 前端: 返回AI回答

3. 查看汤底流程：
   前端 → 后端: GET /api/game/solution/{sessionId}
   后端 → 数据库: 查询汤底和对话历史
   后端 → 前端: 返回汤底和游戏统计

4. 用户认证流程：
   前端 → 后端: POST /api/auth/login {credentials}
   后端 → 数据库: 验证用户信息
   后端 → 前端: 返回JWT令牌和用户信息

前端技术栈：
- React 18 + TypeScript
- Vite 构建工具
- Tailwind CSS 样式框架
- React Router 路由管理
- Zustand/Redux 状态管理
- Axios/Fetch API 通信

后端技术栈：
- Node.js + Express.js
- TypeScript
- PostgreSQL/MongoDB 数据库
- Prisma/TypeORM 数据库ORM
- JWT 用户认证
- Winston 日志记录

AI集成：
- DeepSeek API
- 自定义提示工程模板
- 回答验证和过滤机制
- 请求重试和错误处理

部署和运维：
- Vercel (前端部署)
- Railway/Render (后端部署)
- GitHub Actions CI/CD
- Sentry 错误监控
- 环境变量管理
